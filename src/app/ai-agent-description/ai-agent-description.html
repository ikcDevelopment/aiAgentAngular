Agentes de IA — ¿Qué son y cómo funcionan?

Un agente de IA es un sistema que puede percibir su entorno, tomar decisiones y ejecutar acciones para lograr un objetivo. A diferencia de un simple chatbot, un agente puede usar herramientas, consultar datos externos y mantener contexto a lo largo de una conversación.

Conceptos que debes entender:
  • LLM como cerebro: El modelo de lenguaje (GPT, Gemini, Claude, etc.) es el motor de razonamiento del agente, no el agente completo.
  • Tools / Herramientas: Funciones que el agente puede llamar para obtener información o ejecutar acciones (consultar una base de datos, llamar una API, etc.).
  • System Prompt: Las instrucciones base que definen el comportamiento, personalidad y límites del agente.
  • Context Window: La memoria de corto plazo del LLM. Todo lo que el agente "recuerda" en una conversación vive aquí.

Orquestación Multi-Agente

Un sistema multi-agente es una arquitectura donde múltiples agentes especializados trabajan juntos, coordinados por un agente orquestador que decide quién atiende cada solicitud.

Piénsalo así: el orquestador es el gerente, y los agentes especializados son los empleados con habilidades específicas. El gerente recibe la solicitud del cliente y la delega al empleado correcto.

Patrones que debes conocer:
  • Router / Orquestador: Analiza la intención del usuario y rutea al agente correcto.
  • Agente Especialista: Resuelve un tipo específico de solicitud con sus propias herramientas y contexto.
  • Agente de Validación: Verifica que se tienen todos los datos necesarios antes de procesar una solicitud.
  • Agente Genérico / Fallback: Maneja solicitudes que no corresponden a ningún especialista.

RAG — Retrieval Augmented Generation

RAG es una técnica que permite a un LLM responder preguntas usando información externa sin necesidad de reentrenar el modelo. En lugar de depender solo de lo que aprendió durante el entrenamiento, el agente busca información relevante en una fuente de datos y la incluye en su contexto antes de responder.

Flujo básico de RAG:
  • Usuario hace una pregunta
  • El sistema busca información relevante en la fuente de datos (JSON, documentos, vector DB)
  • Esa información se incluye en el prompt junto con la pregunta
  • El LLM responde basándose en esa información — no alucina porque tiene los datos reales
